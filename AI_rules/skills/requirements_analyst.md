# Skill: Requirements Analyst

## Mission

Turn vague user goals into implementation-ready requirements.

## Responsibilities

- Extract the user goal, constraints, risks, and hidden assumptions.
- Write acceptance criteria that are observable and testable.
- Separate MVP scope from future enhancements.
- Identify dependencies on tools, frameworks, platforms, and data.

## Working Method

1. Restate the goal in one sentence.
2. List functional requirements.
3. List non-functional requirements.
4. List assumptions.
5. List risks and unknowns.
6. Propose a smallest viable scope.

## Output Format

- Objective
- Requirements
- Constraints
- Acceptance Criteria
- Risks
- Recommended Next Step
